Control Anki reviews with global hotkeys — even when Anki is in the background.

Works on <b>Linux</b> (Wayland / X11) and <b>Windows</b>.

<hr>

<b>Why you need this</b>

Anki's built-in shortcuts only work when the Anki window is focused. This addon lets you answer cards no matter what app is in front — your browser, your IDE, your terminal, anything.

<b>Perfect for stealth studying:</b>
Keep Anki minimized or behind another window. Glance at a card, then answer without ever switching windows. Your colleagues will just see you typing — they'll never know you're doing flashcards. Great for vocabulary, formulas, or anything you want to drill during downtime.

<hr>

<b>Features</b>

- Answer cards from any application without switching windows
- Intercept mode: hotkey combos are consumed and not passed to other apps
- Configurable hotkeys via <b>Tools → Global Hotkeys Config</b>
- Changes take effect instantly — no restart needed
- Linux: automatic snap/flatpak sandbox detection with step-by-step guidance

<b>Default hotkeys</b>

Ctrl+Shift+Space — Show Answer
Ctrl+Shift+1 — Again
Ctrl+Shift+2 — Hard
Ctrl+Shift+3 — Good
Ctrl+Shift+4 — Easy

All hotkeys are fully customizable.

<hr>

<b>Requirements</b>

• <a href="https://ankiweb.net/shared/info/2055492159">AnkiConnect</a> addon (must be installed separately)

<b>Linux:</b>
pip install evdev requests
sudo usermod -a -G input $USER  (then log out and back in)
Native Anki install only — snap/flatpak not supported.

<b>Windows:</b>
pip install keyboard requests
No special permissions needed.

<hr>

<b>How it works</b>

A lightweight daemon runs in the background when Anki is open.
On Linux it reads raw keyboard events via evdev, bypassing the display server entirely (works on both Wayland and X11).
On Windows it registers a system-level keyboard hook via the keyboard library.
When a hotkey fires, it calls the AnkiConnect HTTP API to control your review session.
