在任意应用激活时，通过全局快捷键控制 Anki 复习——无需切换窗口。

支持 <b>Linux</b>（Wayland / X11）和 <b>Windows</b>。

<hr>

<b>为什么需要它？</b>

Anki 自带的快捷键只在 Anki 窗口激活时有效。这个插件让你在任何应用前台时都能答题——浏览器、编辑器、终端……随便什么都行。

<b>摸鱼神器用法：</b>
把 Anki 最小化或藏到其他窗口后面，把卡片做成看起来像正常工作内容的样子（代码注释风格的单词卡、伪装成文档的知识点……），瞄一眼卡片内容，按快捷键答题，全程不切窗口。同事只会看到你在敲键盘，完全不知道你在背单词。上班摸鱼、开会摸鱼、等编译摸鱼，时间全部利用起来。

<hr>

<b>功能特性</b>

- 在任意窗口激活状态下触发复习快捷键
- 拦截模式：热键组合不会传递给其他程序
- 配置界面：<b>Tools → 全局热键配置</b>，修改立即生效，无需重启
- Linux：自动检测 snap / flatpak 沙箱并给出分步引导

<b>默认热键</b>

Ctrl+Shift+Space — 显示答案
Ctrl+Shift+1 — Again（重来）
Ctrl+Shift+2 — Hard（困难）
Ctrl+Shift+3 — Good（良好）
Ctrl+Shift+4 — Easy（简单）

所有热键均可自由修改。

<hr>

<b>安装要求</b>

• 需先安装 <a href="https://ankiweb.net/shared/info/2055492159">AnkiConnect</a> 插件

<b>Linux：</b>
pip install evdev requests
sudo usermod -a -G input $USER（执行后注销重新登录）
仅支持原生安装包，不支持 snap / flatpak。

<b>Windows：</b>
pip install keyboard requests
无需任何权限配置，安装即用。

<hr>

<b>工作原理</b>

Anki 启动时自动拉起一个轻量后台 daemon。
Linux 上通过 evdev 直接读取原始键盘事件，完全绕过显示服务器，Wayland 和 X11 均支持。
Windows 上通过 keyboard 库注册系统级键盘钩子。
触发热键后调用 AnkiConnect HTTP API 控制复习流程，Anki 关闭时 daemon 自动退出。
